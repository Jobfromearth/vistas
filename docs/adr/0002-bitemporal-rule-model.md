# 双时间轴的规则版本模型

每条 RuleChunk 携带两根独立的时间轴:**观察区间**(observed_from/to,流水线自动产出,必有)与**法律生效区间**(valid_from/to,仅在可被可靠提取时才填,如 SFS 修正案的 ikraftträdande 日期;提取不出即留空)。tool 返回必须明示使用的是哪根轴:法律源答"自 X 日起生效",指南源答"页面于 X 日前后更新"。

## 为什么不用单轴

把 diff 观察到变更的日期当作 valid_from(v2 计划的隐含做法)会让时间线系统性地偏差数周——机构页面更新滞后于法规生效,抓取又滞后于页面更新——且事后无法区分修复。时间线是本项目宣称最值钱的能力,对它说谎的代价最高。

## Consequences

- rule_timeline / recent_changes 的返回 schema 需区分两轴;指南源的时间线只能诚实地表述为"内容变更观察记录"。
- 法律生效日期的提取(从 SFS 修正案头部)是 M1 解析器的必做项,不是增强项。
