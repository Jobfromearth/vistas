# Vistas — 瑞典移民与求职法规助手 · 项目计划书(v3 · 本地优先)

> 工作名 **Vistas**:瑞典语 vistas = 居留/停留,恰好也是英文单词(远景)。可换,如 Beviljad(已批准)、Uppehåll(居留)。
>
> 一句话定位:**面向 agent 生态的瑞典移民法规数据服务——每个回答都带官方出处和"该规则何时变过"的时间线,以本地 MCP 包 + 开放数据快照分发,不做自有前端、不设服务器。**
>
> v3 变更(经拷问会话逐条定案,详见 `docs/adr/`):真实用户优先;**本地优先分发,远程延后**(ADR-0001);**双时间轴**规则模型(ADR-0002);**词法检索为内核,翻译交给调用方 agent**(ADR-0003);**数据集完全开放**(ADR-0004);技术栈相应崩缩为"流水线 + stdio server"两块。领域术语见 `CONTEXT.md`。

---

## 1. 愿景与边界

**首要目标**:真实用户优先——让非欧盟背景的在瑞学生、毕业生、工签持有者真正用起来;作品集叙事是副产品。早期用户明确接受为"已经在用 Claude Code / Codex 等 agent 的留学生"这一小众精准群体。

**覆盖节奏**:最终承诺四类人群全覆盖(学生、毕业生、工签持有者、跨境通勤者);**上线时覆盖签证类问题**(学签延期、求职签、学签转工签、工签、PUT、家属签——前三类人群的交集),跨境税务(SINK)留 M6。

**核心问题**:签证规则分散在多个机构、多种语言、多个版本中,且频繁变更(如工资门槛调整)。LLM 对这些规则的记忆一律过时且无出处——Vistas 让任何 agent 获得"带引用锚点 + 时间线"的可靠回答能力。

**明确的边界(第一天就写进 tool 描述)**:
- 不是法律建议(juridisk rådgivning),是**带出处的法规数据服务**
- 每个 tool 返回强制携带**分层引用锚点**:法律源到条款级(kap. + §),指南源到页面+章节级,返回中明示层级;给不出锚点就返回明确的"无数据"
- 不处理个案判断("我的申请会不会被拒"),只提供规则本身
- **零个人数据是架构事实而非承诺**:本地形态下用户 query 永不离开用户机器;画像仅作为 tool 参数即用即弃

**为什么值得做**:开源世界没有瑞典特定的同类项目(已调研);规则版本化时间线这个能力连商业渠道都罕见;MCP 生态里"垂直领域权威数据源"类 server 是刚起步的空缺;而你自己就是目标用户,需求判断零距离。

## 2. 数据层(项目的地基,也是最大工作量)

### 2.1 数据源清单(按优先级)

| 优先级 | 来源 | 内容 | 获取方式 |
|--------|------|------|----------|
| P0 | Riksdagen 开放数据 API | 法律全文(SFS,如 Utlänningslagen 2005:716) | 官方 API,结构化,免爬 |
| P0 | Migrationsverket 网站 | 各类签证指南页(sv/en 双语) | 礼貌抓取 |
| P0 | Migrationsverket 法律立场文件 | rättsliga ställningstaganden PDF | 直接下载 |
| P1 | Migrationsverket 统计与处理时间页 | 排队时间、批准率 | 抓取,高频更新 |
| P2 | Skatteverket | SINK、跨境通勤税务 | 抓取(M6) |
| P2 | Försäkringskassan | 社保关联规则 | 抓取(M6) |
| P2 | Lifos 数据库 | 国别信息与判例摘要 | 公开发布物(M6) |

### 2.2 合规(红线,按形态划分)

本地优先形态把合规面收缩为一件事:**公开快照里放什么**。

- **抓取本身**:瑞典著作权法(1960:729)9 § 下法律文本与机构决定不受版权保护;瑞典落实 EU 开放数据指令(öppna data-lagen 2022:818)。工程礼仪:遵守 robots.txt;User-Agent 带项目名与联系邮箱;≤1 请求/秒;条件请求(ETag/Last-Modified)做增量,绝不反复全量抓
- **快照再分发(ADR-0004)**:法律源全文打包无争议。指南源(Migrationsverket 散文内容)在**逐站核实 villkor 之前**,公开快照只含摘录 + 结构化事实(门槛数字、日期——事实本身无版权)+ 锚点链接;流水线内部照存全文做 diff。**核实 villkor + 署名要求入 schema 是 M1 入库前置门禁**
- **数据许可**:快照以 CC0 或 CC-BY 发布(视 villkor 核实结果),首次发布前定案,发布后不可撤回
- **GDPR**:本地形态下不适用(不处理任何用户个人数据);**若未来启用远程形态**,按"隐私由设计"执行:内存限流不落盘 IP、可观测层只记元数据不存 query 原文、访问日志 ≤7 天轮转、配隐私声明
- **零个人内容入库**:不抓案例、不抓姓名

### 2.3 规则版本化(核心能力)

每个入库的规则单元(RuleChunk,见 `CONTEXT.md`)携带**双时间轴**(ADR-0002):

```
RuleChunk {
  content, language,                     # sv / en
  source_type,                           # legal | guidance
  anchor,                                # 法律源: kap.+§ / 指南源: URL+章节标题
  observed_from, observed_to,            # 观察区间:流水线自动产出,必有
  valid_from, valid_to,                  # 法律生效区间:仅可靠提取时填(SFS ikraftträdande),否则留空
  supersedes                             # 版本链:指向被替代的旧版本
}
```

- 每日增量抓取 → diff 检测变更 → 旧版本关闭区间、新版本开启区间,**永不删除旧版**
- tool 返回明示时间轴:法律源答"自 X 日起生效";指南源答"页面于 X 日前后更新"——**永不用观察日期冒充生效日期**
- 由此获得:变更日志、时间线回答、"你看到的旧攻略已过时"的对比能力
- 数据集完全开放(ADR-0004):壁垒不是数据不可复制,而是持续运转的流水线与维护信誉

### 2.4 解析

- 快速通道:markitdown 处理 HTML 与规整文档
- 重型通道:MinerU 或解析 API 处理复杂 PDF(立场文件、表格密集页)
- **法律生效日期提取**(SFS 修正案头部)是解析器必做项,双时间轴依赖它
- 双语对齐:同一页面的 sv/en 版本互相关联,作为翻译对照与一致性校验素材

## 3. 检索层(词法内核,ADR-0003)

1. **词法召回**:SQLite FTS5(BM25)+ 瑞/英同义词表 + 按签证类型/主题的结构化索引。语料总量数千 chunk,不带 embedding 模型
2. **跨语言交给调用方**:tool 描述与 SKILL.md 要求 query 参数为瑞典语/英语关键词;用户的中文/任意语言问题由 agent 翻译改写后再调用
3. **画像过滤**:适用性路由作为 tool 参数实现(见 4.1),学生问延签不会召回难民条款
4. **时间过滤**:默认只召回现行规则(observed_to = null);`as_of_date` 参数放开历史区间
5. **父子文档**:条款小块命中 → 调出所在章节全文返回
6. **强制引用**:所有 tool 返回均为结构化结果(规则原文 + 分层锚点 + 官方链接 + 双时间轴 + 快照构建时间),不做自由生成——**生成留给调用方 agent,Vistas 只保证数据可靠**
7. **升级路径**:若评测证明词法召回不够,加可选的小型量化 embedding(ONNX,几十 MB),不回到 2GB 模型

> 架构决策:query 翻译改写、答案组织、多轮对话全部交给上层 agent(它们本来就擅长),Vistas 收缩为"权威检索层"。责任边界干净:agent 答错措辞是 agent 的事,数据错、出处错才是 Vistas 的 bug。

## 4. MCP server(交付形态)

### 4.1 Tool 设计(第一版收敛到 4 个)

| Tool | 参数 | 返回 |
|------|------|------|
| `search_rules` | query(sv/en 关键词), profile(可选:eu/non_eu, student/worker/graduate, family), language, as_of_date(可选) | 命中规则列表:原文 + 分层锚点 + 官方链接 + 双时间轴 + 快照构建时间 |
| `rule_timeline` | rule_id 或 topic | 该规则的历史版本链:每版内容摘要 + 时间轴(明示观察/生效)+ 变更点 diff |
| `recent_changes` | since(日期), area(可选:work_permit/study/put/…) | 期间内规则变更列表 |
| `get_source` | anchor(分层锚点) | 锚点所在章节全文(父块),供 agent 深读 |

Tool 描述里写死边界声明(非法律建议、仅规则不判个案),这是 MCP 形态下免责声明的落点。

### 4.2 部署形态(ADR-0001)

- **本地 stdio(主,唯一上线形态)**:`uvx vistas-mcp` 一条命令本地跑,自带只读 SQLite 快照;启动时检查 GitHub Releases 有无新快照,有则提示/自动更新。用户 query 永不离开本机
- **快照分发**:GitHub Releases,开放许可,任何人可下载、可整体复制
- **远程 streamable HTTP(延后,不在 M1–M5)**:保留在路线图;若上线后出现真实需求信号(如非开发者用户反馈),同一检索内核加 HTTP 壳,届时按 2.2 的"隐私由设计"补齐 GDPR

### 4.3 SKILL.md(薄壳)

一份教 agent 高效使用 Vistas 的技能文件,内容四块:
1. 什么问题该调哪个 tool(含画像参数怎么填)
2. **query 必须先翻译/改写为瑞典语或英语关键词**(附高频术语对照:工签 → arbetstillstånd 等)
3. 如何向用户呈现引用与时间轴(格式约定:明示"生效"还是"页面更新")
4. 边界与拒答话术(个案判断 → 指引 Migrationsverket 官方渠道)

维护成本约等于零,但让接入体验从"agent 自己摸索"变成"开箱即用"。

## 5. 流水线与运维

不再有常驻服务。系统只有两块:

- **数据流水线(GitHub Actions cron,每日)**:增量抓取 → 解析 → diff → 版本链更新 → 构建 SQLite 快照 → 发布 GitHub Releases。构建日志公开透明,配合带联系邮箱的 UA,抓取行为可审计。若实测 Actions 云段 IP 被源站限流,切 VPS cron 推送(候补方案,成本低,不提前上)
- **MCP server(用户机器,stdio)**:FastMCP,只读快照,零网络依赖(除启动时的更新检查,可关)
- **抓取健康监控**:流水线内解析成功率断言,官网改版 → Actions 报红即告警

**新鲜度承诺**:每个 tool 返回携带快照构建时间,让 agent 能向用户如实转述数据新鲜度。

## 6. 评测体系

| 维度 | 方法 | 运行时机 |
|------|------|----------|
| 检索正确性 | 从官方 FAQ 页反向构造 QA 对 + 人工精标 50–100 条高频问题,验证 top-k 命中;**含"中文原问 → 翻译 → 检索"端到端用例**(ADR-0003 的核心赌注) | CI 门禁 |
| 引用准确率 | 规则脚本校验:返回锚点真实存在、内容一致 | CI 门禁 |
| 版本链正确性 | 注入模拟规则变更 → 验证 diff 检测、双轴区间开闭、supersedes 链完整 | CI 门禁 |
| 无数据行为 | 构造超范围问题集,验证返回明确"无数据"而非硬凑 | CI |
| 端到端质量 | 用 Claude 作为调用方 agent 跑完整问答,LLM-as-Judge 评最终回答(含 SKILL.md 是否被正确遵循) | 夜间/手动,不卡合并 |
| 多语一致性 | 同一问题 sv/en/zh 检索,比对命中规则集合的一致性 | 夜间/手动 |
| 响应速度 | tool 调用 P50/P90(本地快照,预期毫秒级) | CI |

CI 门禁只含纯脚本校验的三项核心(检索正确性 + 引用准确率 + 版本链正确性),回归则阻断合并;LLM-as-Judge 类评测不进门禁(省 API 成本与密钥面)。

| 阶段 | 内容 | 完成标志 |

| M1 数据打通 | **前置门禁:逐站核实 villkor、定快照许可、署名要求入 schema**;Riksdagen API + Migrationsverket 签证板块抓取、解析(含 SFS 生效日期提取)、双轴版本化入库 | 一条命令全量构建 + 增量更新跑通 |

| M4 评测与门禁 | 七维评测 + CI 三项门禁 + Actions 流水线定时化 | 评测报告 v1,流水线连续 7 天自动跑通 |
| M5 发布 | PyPI 发包 + 快照发 Releases + 提交 MCP 目录/registry + 接入教程(一页图文) | 有第一个非本人安装者 |

## 8. 风险与对策

- **答错的后果真实存在** → 结构化返回不自由生成 + 强制分层锚点 + 双轴不冒充 + 无数据即明说 + tool 描述内置边界声明,五道防线全是硬约束
- **agent 翻译质量拖累召回**(ADR-0003 的赌注)→ 评测集含中文端到端用例;SKILL.md 附术语对照表;不够再上小型 ONNX embedding
- **调用方 agent 乱发挥** → SKILL.md 约定呈现格式;端到端评测覆盖"agent 拿到正确数据后是否答对"
- **官方网站改版导致抓取失效** → 解析器按源站模块化,流水线断言解析成功率,失败即 Actions 报红
- **Actions 云段 IP 被源站限流** → 切 VPS cron 候补,切换成本低
- **指南源 villkor 不允许全文再分发** → 保守姿态已是默认(摘录+事实+链接),核实通过才放开,无需回撤
- **规则理解偏差(瑞典语法律文本)** → 双语对照校验;高频问题人工精标答案作为回归锚点
- **维护成本** → 增量 + 版本化 + Actions 全自动,日常维护压缩为"看告警";数据源按 P0→P2 严格排序,不贪多

## 9. 技术栈

| 层 | 选型 |
|----|------|
| MCP | FastMCP(官方 Python SDK),stdio;远程 HTTP 壳延后 |
| Python 工具链 | uv + ruff + pytest |
| 抓取与解析 | httpx + 条件请求缓存;markitdown(快速通道)+ MinerU / 解析 API(重型通道) |
| 存储 | SQLite(版本库 + FTS5 全文索引),随包分发只读快照 |
| 检索 | FTS5/BM25 + 瑞英同义词表 + 结构化索引;可选升级:小型量化 ONNX embedding |
| 流水线 | GitHub Actions cron(候补:VPS cron) |
| 分发 | PyPI(`uvx vistas-mcp`)+ GitHub Releases(数据快照,开放许可) |
| 评测 | pytest + 自建 runner(CI 门禁)+ LLM-as-Judge(夜间/手动),GitHub Actions |

v2 → v3 砍掉:FastAPI、Postgres+pgvector、Redis、ARQ、Langfuse+OTel、Caddy、docker-compose、VPS 常驻服务、bge-m3+reranker。与 Kollega 共享后端基建的叙事不再成立(ADR-0001):Kollega 证明全栈能力,Vistas 证明**数据工程深度与克制的架构判断**。
